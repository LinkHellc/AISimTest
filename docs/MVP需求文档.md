# 汽车空调热管理需求测试用例生成器 - MVP需求文档

## 1. 产品概述

### 1.1 产品定位
面向汽车空调热管理开发工程师的本地桌面应用，实现从Word需求文档到功能测试用例的自动化生成。

### 1.2 目标用户
- 汽车空调热管理系统开发工程师
- 使用MATLAB/Simulink进行软件开发的测试工程师
- 需要处理Word格式需求文档和CAN/LIN信号矩阵的技术人员

### 1.3 MVP范围
**包含**：Word需求导入、基础测试用例生成、本地配置管理
**不包含**：知识库RAG、DBC/LDF文件解析、复杂需求追溯

## 4. 界面原型（关键页面）

### 4.1 主界面布局
```
┌─────────────────────────────────────────────────────────┐
│  Logo    [需求导入] [信号管理] [用例生成] [设置]          │  ← 顶部导航
├──────────┬──────────────────────────────────────────────┤
│          │                                              │
│ 需求树    │         主内容区域                            │
│ - 需求1   │         - 需求详情/编辑                        │
│   - 1.1  │         - 测试用例列表                         │
│   - 1.2  │         - 生成结果展示                         │
│ - 需求2  │                                              │
│          │                                              │
│ [导入]   │                                              │
│ [导出]   │                                              │
└──────────┴──────────────────────────────────────────────┘
    ↑ 左侧边栏               ↑ 右侧主区域
```

### 4.2 关键交互流程

**流程1：Word导入 → 需求条目化**
```
点击"导入需求" → 选择.docx文件 → 解析中 →
显示解析结果（树形）→ 手动修正（可选）→ 保存
```

**流程2：配置LLM → 生成测试用例**
```
进入设置 → 配置API密钥 → 测试连接 →
选择需求 → 点击"生成用例" → 等待生成 →
展示结果 → 人工编辑 → 导出
```

## 5. 技术约束

### 5.1 技术栈
- **前端**：Tauri + React + TypeScript
- **后端**：Python FastAPI
- **文档解析**：python-docx, pandas/openpyxl
- **LLM集成**：langchain/openai

### 5.2 部署方式
- 本地桌面应用安装包（.msi for Windows）
- 无需服务器部署
- 数据本地存储（SQLite/JSON）

## 6. 验收标准

### 6.1 功能验收
- [ ] 能成功解析至少3种不同格式的Word需求文档
- [ ] 能正确提取需求层级和ID
- [ ] 能配置OpenAI API并成功调用
- [ ] 能生成结构完整的测试用例（步骤、预期结果明确）
- [ ] 能导出符合模板的Excel文件

### 6.2 性能验收
- [ ] 100页Word文档解析 < 5秒
- [ ] 单次生成10条用例 < 60秒
- [ ] 界面无卡顿，操作流畅

### 6.3 用户体验验收
- [ ] 新用户能在5分钟内完成首次用例生成
- [ ] 错误提示清晰，能指导用户解决问题
